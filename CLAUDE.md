## Data Storage Specification （重要）

-   所有使用数据存储在单个本地 JSON 文件 `timetrack_data.json` 中，位于 `%APPDATA%\TimeTrack` 目录（程序启动时自动创建目录）。
-   程序启动时将整个 JSON 文件加载到内存数据结构中，退出时和每5分钟定时回写覆盖文件，防止意外崩溃丢失数据。
-   JSON 结构设计如下（必须严格遵循）：  
    json
    
    {  
      "days": {  
        "2026-05-10": {  
          "totalSeconds": 18540,  
          "apps": \[  
            {  
              "name": "Google Chrome",  
              "processKey": "chrome.exe",  
              "seconds": 10200,  
              "lastUpdate": "2026-05-10T14:20:00"  
            }  
          \]  
        }  
      },  
      "ignoredApps": \[  
        { "processKey": "explorer.exe", "addedTime": "2026-05-09T08:00:00" }  
      \],  
      "settings": {  
        "autoStart": true,  
        "checkIntervalSec": 2,  
        "idleTimeoutMin": 5,  
        "idleDetectionEnabled": true,  
        "dataRetentionDays": 0  
      }  
    }
    
-   -   `days` 的 key 为 "YYYY-MM-DD" 格式的日期字符串。
    -   每个 app 的 `name` 为显示友好名称，`processKey` 用于合并窗口和忽略匹配。
    -   `seconds` 为当日该应用的累计前台秒数。
    -   `settings` 存储用户配置，`dataRetentionDays` 为 0 表示永久保留。
-   跨天运行时，必须在内存中自动切换日期，并在 JSON 中创建新日期的条目。
-   忽略列表应立即反映在计时逻辑中，从内存结构读取，无需每次都查询文件。
-   JSON 解析器必须自行实现，禁止使用任何第三方 JSON 库。解析器需支持基本的字符串转义、Unicode转义（\\uXXXX），以及标准的数字、布尔、null、数组和对象解析。代码应健壮，面对格式错误的数据文件不至于崩溃，可回退到默认空数据结构。

## Core Functional Requirements (必须严格遵守)

### 计时逻辑

1.  仅当窗口获得焦点、非最小化、系统非锁屏/睡眠/屏保、显示器未关闭、并且用户有鼠键操作（若空闲检测开启）时，才累计时间。
2.  每秒检查一次窗口状态（可配置1-10秒），使用高精度计时器（如QueryPerformanceCounter）记录累积秒数，避免时间漂移。
3.  同一进程多个窗口合并为一个应用，友好名称获取逻辑：优先查询产品名称，其次文件描述，最后窗口标题。
4.  每日00:00点强制分割日期，跨天时段自动分配到两天。
5.  程序退出、系统关机时（处理WM\_QUERYENDSESSION）立即保存当前数据，不丢失任何秒数。

### 界面与交互

-   托盘图标常态显示，左键打开面板，右键菜单包含：打开面板、暂停/恢复统计(全局)、设置、退出。
-   退出需要确认对话框（MessageBox），确认后方可退出。
-   主面板设计：
-   -   标题栏显示今天日期和总时长。
    -   列表控件（ListView）：软件名称 | 时长(HH:MM:SS) | 百分比。
    -   列表项支持右键菜单：添加到忽略列表。
    -   日历控件（MonthCal）选择历史日期，界面自动切换显示该日数据。
    -   设置按钮打开设置子窗口：自启开关、检测频率滑条（1-10）、空闲阈值（1-30分钟）、数据清理期限（天数下拉）。
    -   忽略列表管理按钮：打开列表窗口，支持移除选中项。

### 忽略功能

-   添加应用后立即从当前窗口停止计时，之前累计的数据保留在历史中。
-   从忽略列表移除后，下一检测周期恢复计时，不补录之前忽略期间的时长。
