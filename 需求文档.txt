📋 产品需求文档 (PRD) – 电脑应用使用时长统计工具
1. 文档信息
项目名称：TimeTrack （暂定）

产品定位：轻量级本地电脑应用使用时长统计工具

目标用户：希望了解自己电脑使用习惯、进行时间管理的个人用户

版本：V1.0

2. 产品目标
自动、无感地记录用户每天前台应用的使用时长，提供直观的数据回顾和基本的过滤设置，帮助用户提升时间感知，不做任何数据上传。

3. 功能需求
3.1 核心统计模块
编号	功能	详细说明
F-01	开机自启与后台运行	软件默认开启开机自启；启动后直接进入系统托盘，不弹出主界面；重复启动时激活已运行实例。
F-02	前台窗口监测	以当前获得输入焦点的前台窗口为统计对象；窗口最小化、失去焦点、系统锁屏/睡眠/屏保/关显示器时暂停计时。
F-03	空闲检测	提供可关闭的空闲判定：连续无键盘/鼠标操作超过 N 分钟后暂停计时（默认5分钟），恢复操作后继续。
F-04	应用识别与合并	同一软件多个窗口合并为一个应用统计，显示友好名称（如“Google Chrome”）；无法获取时降级使用进程名或窗口标题。
F-05	按自然日分界	以每日 00:00 为分界点，跨天运行时长自动切分到相应日期，历史日数据封存不可变。
3.2 系统托盘交互
编号	功能	详细说明
F-06	托盘图标	程序运行时常驻系统托盘区，图标可正常态显示。
F-07	左键单击	打开统计面板（主界面）。
F-08	右键菜单	包含：打开统计面板、暂停/恢复统计、设置、退出程序。
F-09	退出确认	退出时弹窗确认（可在设置中关闭），防止误触。
3.3 数据查看
编号	功能	详细说明
F-10	今日概览	默认显示当日数据：顶部今日总使用时长；应用按使用时长降序列表，显示名称、时长(HH:MM:SS)、占比(%)。正在使用的应用标注“计时中”并动态刷新。
F-11	历史记录	通过日历选择日期查看过去任意天的详情，布局同今日，支持前后日快速切换，无记录日期置灰。
F-12	时间显示规则	不足1分钟按1分钟计；显示格式：“X小时Y分钟”；历史视图中可查看总分钟数。
3.4 忽略/过滤
编号	功能	详细说明
F-13	预置忽略列表	系统自带推荐忽略清单（自身、系统外壳等），用户可重置。
F-14	添加忽略	支持三种方式：今日列表右键“不再统计”、手动输入名称模糊添加、从当前运行列表选择添加。添加后立即停止计时，历史数据保留。
F-15	忽略管理	查看/移除/清空忽略列表；移除后从当前时刻恢复计时，不补录。
3.5 设置
编号	功能	详细说明
F-16	开机自启开关	一键切换，即时生效。
F-17	检测频率	1-10秒可调，默认2秒。
F-18	空闲判定设置	开关与阈值（1-30分钟），默认开启且5分钟。
F-19	数据管理	清除全部历史数据（需确认）；设置自动清理期限（90天/180天/永久）。
F-20	其他	关于页面（版本号）；预留语言选择。
4. 非功能需求
性能：CPU占用在空闲时<1%，检测运行时<3%。

数据安全：所有数据存于本地，无任何网络请求。

兼容性：Windows 10/11 为主，需考虑不同DPI缩放。

可扩展性：统计模块、视图模块、导出模块低耦合，为后续周/月视图、图表、导出扩展预留接口。

5. 数据结构草案
每日使用记录：日期、应用名、应用进程标识(用于合并)、累积秒数、最后更新时间。

忽略列表：应用匹配关键字、添加时间。

设置：键值对保存自启、检测间隔、空闲阈值等。
